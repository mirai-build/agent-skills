# ドメインモデリング一覧

> 使い方: `domain_model` 配下の入口として必ず最初に作成し、どの順で設計を深めるかを一覧で把握できるようにする。詳細ドキュメントを追加したら、この一覧も更新する。

## 0. 文書の目的

この文書は、ドメインモデリング配下の設計を「一覧（概略）→ 詳細」の順で読めるようにし、`01_system_purpose` `02_actors` `03_usecases` `04_journeys` `05_bounded_contexts` の整備順を共有するための正本である。

## 1. 整備順

| 段階 | ディレクトリ | 役割 |
| --- | --- | --- |
| 1 | `01_system_purpose/` | システムの目的と価値の整理 |
| 2 | `02_actors/` | actor の定義と責務の整理 |
| 3 | `03_usecases/` | ユースケースの整理 |
| 4 | `04_journeys/` | ユースケースをつないだ一連の流れの整理 |
| 5 | `05_bounded_contexts/` | bounded context 分割の整理 |

## 2. ドキュメント案内

| ドキュメント | 役割 | 更新タイミング |
| --- | --- | --- |
| `01_system_purpose/00_overview.md` | システムの目的配下の入口 | 目的やスコープの理解が更新されたとき |
| `02_actors/00_overview.md` | actor 定義配下の入口 | actor の追加や責務の見直しが入ったとき |
| `03_usecases/00_overview.md` | ユースケース配下の入口 | ユースケースの追加や変更が入ったとき |
| `04_journeys/00_overview.md` | ジャーニー配下の入口 | ユースケースをまたぐ流れを整理したいとき |
| `05_bounded_contexts/00_overview.md` | bounded context 分割配下の入口 | 境界や関係性の見直しが入ったとき |
| `99_glossary.md` | 用語の正本。日本語名とシステム英語名をそろえる | 用語が増えたとき、意味が揺れたとき |
